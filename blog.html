Important FAQ

What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of an HTML or XML document as a tree-like hierarchy of objects, where each element, attribute, and piece of text in the document is represented as a node in the tree.

The DOM allows programming languages like JavaScript to access and manipulate the content, structure, and style of a web page dynamically. Through the DOM, developers can add, remove, or modify HTML elements and their content, change CSS styles and classes, respond to user events (clicks, key presses, etc.), and create interactive and dynamic web applications.

The DOM is platform and language-independent, meaning it can be used with any programming language that supports it, though it's most commonly used with JavaScript in web browsers.

How do you select an element from the DOM?

There are several methods to select elements from the DOM in JavaScript:

getElementById() - Selects a single element by its ID attribute

getElementsByClassName() - Selects all elements with a specific class name

getElementsByTagName() - Selects all elements with a specific tag name

querySelector() - Selects the first element that matches a CSS selector

querySelectorAll() - Selects all elements that match a CSS selector

Modern best practice is to use querySelector() and querySelectorAll() as they accept any valid CSS selector, making them more flexible and powerful than the older methods.

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in JavaScript where instead of adding event listeners to individual child elements, you add a single event listener to a parent element. This parent listener can handle events from all its child elements through event bubbling.

When an event occurs on a child element, it "bubbles up" through its parent elements. By attaching a listener to the parent, you can catch these bubbled events and determine which child element triggered them using the event.target property.

Why is it useful?

  • Better Performance: Fewer event listeners mean less memory usage and better performance
  • Dynamic Elements: Works automatically with elements added to the DOM after page load
  • Cleaner Code: One event listener instead of many makes code easier to maintain
  • Memory Efficiency: Reduces memory footprint since you're not creating multiple function references
How do you manipulate an element's attributes and styles using the DOM?

The DOM provides multiple methods and properties to manipulate element attributes and styles dynamically.

Manipulating Attributes:

  • getAttribute() / setAttribute() - Get or set attribute values
  • removeAttribute() - Remove an attribute
  • Direct property access - element.id, element.className, element.href, etc.

Manipulating Styles:

  • element.style.property - Set inline styles directly (e.g., element.style.color = 'red')
  • element.style.cssText - Set multiple styles at once
  • classList - Add, remove, or toggle CSS classes (element.classList.add('active'))
  • getComputedStyle() - Get the actual computed styles of an element

Using classList methods (add, remove, toggle, contains) is generally preferred over directly manipulating the className property as it provides better control and avoids accidentally removing other classes.
